# Simple ESC I²C Interface Contract

## Purpose

This is a small I²C control/telemetry interface for a hobby STM32 ESC running code generated by STM32 Motor Control Workbench / MCSDK.

The ESC MCU still runs all motor-control logic locally. The host MCU only sends simple commands and reads status/telemetry.

## Design goals

* Simple enough to implement quickly.
* No floats on the bus.
* Short fixed-size messages.
* Safe enough for a noisy motor board.
* Easy to debug with a logic analyser.
* No dependency on a generic I²C library.

## Bus defaults

| Item            |                   Value |
| --------------- | ----------------------: |
| ESC role        |      I²C target / slave |
| Host role       | I²C controller / master |
| Default address |                  `0x34` |
| Bus speed       |                 100 kHz |
| Max block size  |                32 bytes |
| Endian          |           little-endian |
| Floats          |                      no |
| CRC             |  not required initially |

Use scaled integers:

| Quantity    |  Type |          Unit |                    Example |
| ----------- | ----: | ------------: | -------------------------: |
| Voltage     | `u32` |            mV |         `24375` = 24.375 V |
| Current     | `i32` |            mA |         `-1200` = -1.200 A |
| Temperature | `i32` |           m°C |        `45625` = 45.625 °C |
| Speed       | `i32` |           dHz | `500` = 50.0 Hz = 3000 rpm |
| Duty        | `i16` | percent × 100 |            `5234` = 52.34% |

## Register access model

Use a normal register-pointer model.

Write:

```text
[i2c_addr+w] [register] [payload...]
```

Read:

```text
[i2c_addr+w] [register]
repeated-start
[i2c_addr+r] [payload...]
```

The first byte of a write is the register address. Any following bytes are the payload for that register.

## Register map

| Register | Name        | Access |     Size | Purpose                       |
| -------: | ----------- | ------ | -------: | ----------------------------- |
|   `0x00` | `ID`        | read   |  8 bytes | Version and capabilities      |
|   `0x10` | `STATUS`    | read   | 16 bytes | State, faults, command result |
|   `0x20` | `COMMAND`   | write  | 16 bytes | Send command to ESC           |
|   `0x30` | `TELEMETRY` | read   | 32 bytes | Main live telemetry           |

That is the whole required interface.

## `ID` register, `0x00`, 8 bytes

| Offset | Field           |  Type | Notes            |
| -----: | --------------- | ----: | ---------------- |
|      0 | `proto_major`   |  `u8` | Start at `1`     |
|      1 | `proto_minor`   |  `u8` | Start at `0`     |
|      2 | `fw_major`      |  `u8` | Firmware version |
|      3 | `fw_minor`      |  `u8` | Firmware version |
|      4 | `hw_id`         |  `u8` | Board variant    |
|      5 | `max_block_len` |  `u8` | `32`             |
|      6 | `capabilities`  | `u16` | Feature bits     |

Suggested capability bits:

| Bit | Meaning                           |
| --: | --------------------------------- |
|   0 | speed command supported           |
|   1 | duty command supported            |
|   2 | current measurement available     |
|   3 | temperature measurement available |
|   4 | reverse supported                 |
|   5 | brake supported                   |

## `STATUS` register, `0x10`, 16 bytes

| Offset | Field              |  Type | Notes                                |
| -----: | ------------------ | ----: | ------------------------------------ |
|      0 | `seq`              |  `u8` | Increments whenever status updates   |
|      1 | `esc_state`        |  `u8` | Simple interface state               |
|      2 | `mc_state`         |  `u8` | MCSDK motor state, raw if convenient |
|      3 | `last_cmd_seq`     |  `u8` | Last accepted command sequence       |
|      4 | `last_cmd_error`   |  `u8` | Result of last command               |
|      5 | `reserved`         |  `u8` | zero                                 |
|      6 | `current_faults`   | `u16` | Active faults                        |
|      8 | `occurred_faults`  | `u16` | Latched/occurred faults              |
|     10 | `status_flags`     | `u16` | Summary flags                        |
|     12 | `watchdog_ms_left` | `u16` | Optional, zero if unused             |
|     14 | `reserved`         | `u16` | zero                                 |

Suggested `esc_state` values:

| Value | State    |
| ----: | -------- |
|     0 | boot     |
|     1 | idle     |
|     2 | running  |
|     3 | stopping |
|     4 | fault    |

Suggested `last_cmd_error` values:

| Value | Meaning                |
| ----: | ---------------------- |
|     0 | OK                     |
|     1 | unknown opcode         |
|     2 | invalid state          |
|     3 | parameter out of range |
|     4 | motor fault active     |
|     5 | busy                   |
|     6 | bad length             |

Suggested `status_flags` bits:

| Bit | Meaning                   |
| --: | ------------------------- |
|   0 | fault present             |
|   1 | running                   |
|   2 | watchdog expired          |
|   3 | undervoltage              |
|   4 | overvoltage               |
|   5 | overtemperature           |
|   6 | overcurrent               |
|   7 | speed feedback unreliable |

## `COMMAND` register, `0x20`, 16 bytes

The host writes this whole block to send a command.

| Offset | Field      |  Type | Notes                          |
| -----: | ---------- | ----: | ------------------------------ |
|      0 | `seq`      |  `u8` | Host increments each command   |
|      1 | `opcode`   |  `u8` | Command code                   |
|      2 | `flags`    |  `u8` | Optional flags, zero initially |
|      3 | `reserved` |  `u8` | zero                           |
|      4 | `param0`   | `i32` | Command-specific               |
|      8 | `param1`   | `i32` | Command-specific               |
|     12 | `param2`   | `i32` | Command-specific               |

No CRC initially. If the bus later proves noisy, add a CRC byte by replacing one reserved or parameter byte.

The ESC should copy the command in the I²C callback, then execute it later in the main loop/task.

## Commands

| Opcode | Name             |          `param0` |      `param1` | `param2` | Notes                        |
| -----: | ---------------- | ----------------: | ------------: | -------: | ---------------------------- |
| `0x00` | `NOP`            |           ignored |       ignored |  ignored | Ping / watchdog refresh      |
| `0x01` | `START`          |           ignored |       ignored |  ignored | Start motor if safe          |
| `0x02` | `STOP`           |           ignored |       ignored |  ignored | Stop/coast                   |
| `0x03` | `CLEAR_FAULTS`   |           ignored |       ignored |  ignored | Acknowledge faults           |
| `0x04` | `SET_SPEED_RAMP` | target speed, dHz | ramp time, ms |  ignored | Main speed command           |
| `0x05` | `ESTOP`          |       reason code |       ignored |  ignored | Immediate stop + latch fault |

Optional later commands:

| Opcode | Name           | Notes                                                          |
| -----: | -------------- | -------------------------------------------------------------- |
| `0x06` | `SET_DUTY`     | Only if the MCSDK project exposes a suitable duty/voltage mode |
| `0x07` | `SET_WATCHDOG` | Only if you need host-loss detection                           |
| `0x08` | `SET_LIMITS`   | Only once basic control is working                             |

## `TELEMETRY` register, `0x30`, 32 bytes

Read this at 10–50 Hz.

| Offset | Field            |  Type |    Unit | Notes                                       |
| -----: | ---------------- | ----: | ------: | ------------------------------------------- |
|      0 | `seq`            |  `u8` |       - | Increments after telemetry snapshot updates |
|      1 | `esc_state`      |  `u8` |       - | Same enum as status                         |
|      2 | `mc_state`       |  `u8` |       - | MCSDK state                                 |
|      3 | `last_cmd_error` |  `u8` |       - | Last command result                         |
|      4 | `status_flags`   | `u16` |       - | Summary flags                               |
|      6 | `current_faults` | `u16` |       - | Active faults                               |
|      8 | `vbus_mV`        | `u32` |      mV | Bus voltage                                 |
|     12 | `ibus_mA`        | `i32` |      mA | Use zero if unavailable                     |
|     16 | `speed_dHz`      | `i32` |     dHz | Mechanical speed; rpm = dHz × 6             |
|     20 | `duty_centi_pct` | `i16` | % × 100 | `5234` = 52.34%                             |
|     22 | `temp_mC`        | `i32` |     m°C | Use hottest measured temp                   |
|     26 | `last_cmd_seq`   |  `u8` |       - | Last accepted command sequence              |
|     27 | `reserved`       |  `u8` |       - | zero                                        |
|     28 | `uptime_s`       | `u32` |       s | Wraps naturally                             |

If bus current or temperature is not available yet, return zero and clear the relevant capability bit.

## Firmware structure

Do this:

```text
I²C callback:
  - receive bytes
  - copy command/register data into a buffer
  - set a flag
  - return quickly

main loop/task:
  - check command_pending
  - validate command length/opcode/state/parameters
  - call MCSDK API
  - update status and last_cmd_error

periodic telemetry task:
  - read MCSDK state/speed/faults
  - read voltage/current/temp if available
  - copy into telemetry struct
  - increment telemetry seq
```

Do not call MCSDK motor-control functions directly inside the I²C interrupt/callback.

## Likely MCSDK API mapping

| I²C action         | MCSDK-side call                                      |
| ------------------ | ---------------------------------------------------- |
| start              | `MC_StartMotor1()`                                   |
| stop               | `MC_StopMotor1()`                                    |
| set speed ramp     | `MC_ProgramSpeedRampMotor1(target_dHz, duration_ms)` |
| clear faults       | `MC_AcknowledgeFaultMotor1()`                        |
| active faults      | `MC_GetCurrentFaultsMotor1()`                        |
| occurred faults    | `MC_GetOccurredFaultsMotor1()`                       |
| mechanical speed   | `MC_GetMecSpeedAverageMotor1()`                      |
| duty reference     | `MCI_GetDutyCycleRefMotor1()` if available           |
| motor state        | `MCI_GetSTMStateMotor1()`                            |
| command/ramp state | `MC_GetCommandStateMotor1()` if available            |
| ramp complete      | `MC_HasRampCompletedMotor1()` if needed              |

Function names can vary slightly with MCSDK version and generated project options. Use the generated `mc_api.h` / `mc_interface.h` as the source of truth.

## Safety behaviour

Minimum useful safety behaviour:

1. Boot into idle/stopped state.
2. Reject `START` while faults are active.
3. `ESTOP` immediately stops the motor and latches a fault-like state.
4. Bad or unknown commands do not change motor output.
5. `STOP` should work from any non-boot state.
6. Optional watchdog: if no valid command arrives for N ms while running, stop/coast.

## First implementation order

1. Confirm the generated MCSDK project runs the motor without I²C.
2. Add I²C target peripheral in CubeMX.
3. Implement `ID`, `STATUS`, and `TELEMETRY` reads with dummy data.
4. Add real MCSDK state/fault/speed telemetry.
5. Implement `COMMAND` write buffering.
6. Implement `NOP`, `START`, `STOP`, and `CLEAR_FAULTS`.
7. Implement `SET_SPEED_RAMP`.
8. Add bus voltage telemetry.
9. Add temperature/current telemetry if available.
10. Add watchdog only if you actually need host-loss protection.

## Minimal host-side flow

Startup:

```text
read ID
read STATUS
if faults present: send CLEAR_FAULTS
send START
send SET_SPEED_RAMP(target_dHz, ramp_ms)
```

Loop:

```text
read TELEMETRY at 10–50 Hz
check status_flags and current_faults
update display/logging/control logic
```

Stop:

```text
send STOP
confirm TELEMETRY/STATUS reports not running
```
